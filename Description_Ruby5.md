# KEYLEVEL_TRADER_Ruby5 — Полное описание и анализ стратегии

> Версия советника: **4.02** | Файл: `KEYLEVEL_TRADER_Ruby5.mq4`

---

## 🧠 Концепция и идея

Ruby5 — это **режимно-адаптивный мультипарный советник** (работает одновременно на 27 парах × 4 таймфреймах). Ключевая идея — **Chameleon Entry**: перед входом советник определяет тип рынка (тренд / боковик / мёртвый) и использует принципиально разную логику входа для каждого режима.

Все сигналы считаются по **закрытым барам** (`shift >= 1`) — без перерисовки.

Выходы и трейлинг — **stateless ACE-движок** (Adaptive Conviction Exit), унаследованный от Ruby1.

## 🔍 Определение режима рынка (Chameleon Entry)

### Efficiency Ratio (Kaufman ER)
Основной инструмент классификации — **ER без лага** (быстрее ADX):

```
ER = |Close[shift] - Close[shift+period]| / sum(|Close[i] - Close[i+1]|, period)
```

| ER | Режим | Действие |
|---|---|---|
| ATR(1 бар) < `EntryMinATRpips` | Мёртвый рынок | Не входим |
| ER >= `ERTrendLevel` (0.35) | **Тренд** | TrendSignal (pullback к EMA) |
| ER <= `ERRangeLevel` (0.25) | **Боковик** | RangeSignal (фейд краёв канала) |
| Между уровнями | Серая зона | Не входим |

> ⚠️ **Ключевое решение**: ER используется вместо ADX именно потому, что он **опережающий** и не лагирует. Это стоит переносить в любые новые советники.

## 📈 Правила входа (Entry Rules)

### Тренд-режим (TrendSignal)
Условия **BUY**:
1. `+DI > -DI` (направление по ADX-индикатору)
2. `EMA(21) > EMA(50)` — оба EMA подтверждают восходящий тренд
3. `Close > EMA(50)` — цена выше медленной EMA
4. HTF-bias ≥ 0 (на старшем ТФ EMA(21) ≥ EMA(50))
5. **Pullback**: предыдущий бар касался `EMA(21)` снизу (`Low <= EMA(21)`) И закрылся выше (`Close > EMA(21)`)

Условия **SELL** — симметрично.

> **Суть**: вход в откат к быстрой EMA в направлении тренда. Классический pullback-вход, усиленный HTF-фильтром.

### Боковой режим (RangeSignal)
Условия **BUY** (fade нижней границы):
1. Цена в нижних `RangeEdgePct` (20%) диапазона последних 20 баров
2. `RSI(14) <= 35` — перепроданность

Условия **SELL** — симметрично (верхняя граница + RSI >= 65).

> **Суть**: контртрендовый вход от краёв канала с подтверждением RSI.

### Дополнительные фильтры перед входом
- Спред ≤ `MaxSpreadPips` (3 пипа по умолчанию)
- `IsMarketDead()` = false (ATR за последние `N` баров не ниже `VolatilityDeadPips`)
- `SARBetrayalDetected()` = false — SAR уже не "предаёт" направление входа
- ATR-стоп >= `MinStopPips` — рынок достаточно волатилен для реального стопа
- Один бар = один сигнал (кэш по `LastTradeTime[pair][tf]`)

## 🚪 Правила выхода (Exit Rules)

Все выходы — в `SmartExitEngine()`, вызывается **каждый тик** до расчёта новых входов.

### Выход 1 — Emergency Exit (жёсткий стоп)
```
Если текущий P&L <= -EmergencyAtR * R → немедленное закрытие
```
`EmergencyAtR = 0.80` → закрывает, если убыток достиг 80% расстояния риска.

> Работает как backstop если рынок проскочил SL или тейк-профит недостижим.

### Выход 2 — Partial Close + Breakeven
При достижении `PartialAtR * R` (1.0R по умолчанию = достигли 1:1):
- Закрывается **50% позиции**
- SL переносится в безубыток + `BreakevenBufferPips` (2 пипа)

> Реализует принцип "снять половину — торговать остатком бесплатно".

### Выход 3 — Conviction Flip (при прибыли)
Только если позиция в прибыли ≥ `TrailingActivationPips` (12 пипов):
```
ConvictionScore <= -ConvictionExitProfit (-0.70)
```
**ConvictionScore** = среднее трёх нормализованных факторов:
- SAR: +1 если SAR "за" позицию, -1 если "против"
- Моментум: направление наклона (EMA5 - EMA13)
- Trend: доля бычьих/медвежьих баров из последних 8

### Выход 4 — Volatility Dead (при убытке)
Если позиция в минусе и `IsMarketDead()` = true → закрытие.
> Устраняет удержание позиции в замёрзшем рынке.

### Выход 5 — SAR Betrayal (при убытке)
Активируется при убытке ≥ `LossActivationR * R` (0.15R):
- Проверяем последние `SARBetrayalBars` (3) баров
- Если на ≥ 2 барах SAR чётко указывает против позиции → выход

### Выход 6 — Momentum Death (при убытке)
При убытке от -0.15R до -0.60R:
- Если ≥ 3 из последних 4 баров моментум (EMA5 - EMA13) развернулся против позиции → выход

### Выход 7 — Conviction Against (при убытке)
```
ConvictionScore <= -ConvictionExitLoss (-0.50)
```
Тот же ConvictionScore что и в Выходе 3, но срабатывает раньше и при убытке.

## 🎯 Управление позицией (Position Sizing)

### Расчёт лота
```
Risk = Balance * RiskPercent / 100
LotDistance = MAX(stopDist, LotSizingMinStopPips * pip)   ← пол стопа
Lot = Risk / ((LotDistance / TickSize) * TickValue)
Lot = CLAMP(Lot, MinLot, MIN(MaxLot, MaxLotPerTrade))
```

> **FIX-2 (ключевое)**: `LotSizingMinStopPips = 20` пипов — пол для расчёта лота, но НЕ для реального SL. Устраняет раздувание лота при узком ATR-стопе.

### ATR-стоп
```
rawStop = ATR(14, shift=1) / pip * StopATRMult (1.5)
stopPips = CLAMP(rawStop, MinStopPips=12, MaxStopPips=120)
TP = entry ± stopPips * InitialTP_RR (3.0)
```
Соотношение риск/прибыль по умолчанию **1:3**.

## 📉 Elastic ATR Trailing Stop

Трейлинг активируется при прибыли ≥ `TrailingActivationPips` (12 пипов):

```
TrendStrength = (бары в направлении позиции) / TrendStrengthBars   [0..1]
DynamicMult = MultSlow + TrendStrength * (MultFast - MultSlow)
             = 0.7 + strength * (1.2 - 0.7)
TrailingDist = ATR(14) * DynamicMult
```

- При сильном тренде (TrendStrength → 1): мультипликатор → 1.2 (широкий трейлинг, даём тренду дышать)
- При слабом тренде (TrendStrength → 0): мультипликатор → 0.7 (плотный трейлинг, фиксируем быстро)

До частичного закрытия SL **не уходит выше entry** (защита от преждевременного стопа).

## 🛡️ Equity Drawdown Brake (Трёхступенчатая защита)

| Ступень | Триггер | Действие |
|---|---|---|
| **Soft (Оборона)** | DD от пика дня ≥ 2% | Риск × 0.5, ERTrendLevel повышается, HTF форсирован, боковик отключён |
| **Hard (Дневной стоп)** | DD от пика дня ≥ 4% | Блокировка новых входов до конца дня |
| **Total (Многодневная)** | DD от исторического пика ≥ 8% | Блокировка входов до восстановления; сбрасывается с гистерезисом |

> Выходы и трейлинг **работают всегда** — только новые входы блокируются.

## 🌐 Мультипарная и мультитаймфреймовая логика

**27 пар**: мажоры + кроссы JPY/AUD/NZD/GBP/EUR

**4 таймфрейма**: H1, M30, M15, M5

**Tier-1 пары** (EURUSD, GBPUSD и ещё 6) — торгуются на всех 4 ТФ.
**Tier-2 пары** (кроссы) — только H1, M30, M15.

HTF-bias: для M5 → M30, M15 → H1, M30 → H4, H1 → D1.

**Хеджирование**: при наличии позиции в одном направлении, советник может открыть до `MaxHedgeOrders` (1) ордеров в обратном направлении.

## 🔄 Основной цикл (OnTick)

```
1. Сброс дневного состояния если новый день
2. Проверка достижения дневной цели
3. SmartExitEngine() — выходы из открытых позиций
4. ElasticATRTrailing() — обновление трейлинга
5. UpdateEquityBrake() — пересчёт уровней защиты
6. Проверки блокировок (TotalLocked, EntriesLockedForDay, InSession)
7. Цикл по парам и ТФ:
   - Проверка спреда
   - Получение сигнала (кэш по бару)
   - CountTrades (нет дубликатов)
   - ExecuteTrade()
```

## 📊 Анализ стратегии

### ✅ Сильные стороны

1. **Режимность (Chameleon)** — одна из главных инноваций. Рынок не всегда в тренде; разделение тренд/боковик/мёртвый устраняет класс ложных сигналов, типичных для трендовых советников в консолидации.

2. **ER вместо ADX** — Efficiency Ratio реагирует быстрее и без лага. Особенно важно для M5-M15.

3. **Многоуровневые выходы** — ConvictionScore, SAR Betrayal, Momentum Death работают как "умная" страховка, позволяя выходить до того, как SL сработает. Теоретически улучшает средний убыток.

4. **Elastic trailing** — адаптируется к рыночной динамике (сила тренда меняет плотность трейлинга). Лучше фиксированного ATR-трейлинга.

5. **FIX-2: LotSizingMinStopPips** — критически важная защита от раздувания лота при узком ATR-стопе. Без этого советник мог открывать огромные позиции в тихом рынке.

6. **Partial Close + Breakeven** — классический механизм "R-нейтрализации", снижает психологическую нагрузку и уменьшает итоговые просадки.

### ⚠️ Потенциальные слабые места

1. **27 пар × 4 ТФ = до 108 комбинаций** — высокая нагрузка на тестер/терминал. Возможны задержки при большом количестве открытых ордеров.

2. **Хеджирование** (`MaxHedgeOrders = 1`) — в ряде стратегий хедж без чёткой логики снятия создаёт нетто-нейтральные позиции, которые просто удваивают комиссию/своп.

3. **ConvictionScore** — синтетический индикатор из 3 факторов с равными весами. Нет бэктестированного доказательства, что такая формула оптимальна. Чувствителен к параметрам.

4. **Мёртвый рынок через ATR** — `VolatilityDeadPips = 4` могут срабатывать по-разному на разных парах (JPY пары vs EURUSD). Лучше бы параметр был процентным от ATR-среднего.

5. **Отсутствие фильтра новостей** — при выходе NFP/ЦБ спред расширяется, MaxSpreadPips=3 должен защищать, но не полностью.

6. **ERTrendLevel = 0.35 / ERRangeLevel = 0.25** — "серая зона" [0.25, 0.35] занимает ~28% пространства ER. При волатильном рынке советник будет часто молчать. Это хорошо для точности, плохо для частоты сделок.
